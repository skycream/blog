# 블로그 자동 생성 가이드라인

## 개요

키워드 입력 → 최종 HTML 블로그 출력까지의 전체 파이프라인 및 작성 규칙

---

## 1. 파이프라인 흐름

```
[입력] 키워드 (예: "역류성식도염", "오렌지주스", "러닝")
                    ↓
┌─────────────────────────────────────────────────────────────┐
│  STEP 1: 경쟁 블로그 분석                                    │
│  ─────────────────────────────────────────────────────────  │
│  • 한글 키워드로 네이버/구글 블로그 검색 (10~15개)            │
│  • 기존 블로그들의 공통 내용 추출                             │
│  • 목적: "뻔한 내용"이 뭔지 파악하여 차별화                   │
└─────────────────────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────────────────┐
│  STEP 2: PubMed 논문 검색                                   │
│  ─────────────────────────────────────────────────────────  │
│  • 영문 키워드로 PubMed 검색                                 │
│  • 최소 50편 이상 수집 (다양한 쿼리 조합 사용)               │
│  • 쿼리 예시:                                                │
│    - "{keyword}"                                            │
│    - "{keyword} treatment"                                  │
│    - "{keyword} diet"                                       │
│    - "{keyword} risk factors"                               │
│    - "{keyword} meta-analysis"                              │
└─────────────────────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────────────────┐
│  STEP 3: 차별화 포인트 추출                                  │
│  ─────────────────────────────────────────────────────────  │
│  • 논문 내용 vs 기존 블로그 비교                             │
│  • 추출 우선순위:                                            │
│    1. 통계/수치가 있는 발견 (예: "32%에서 발견")             │
│    2. 기존 블로그에 없는 새로운 사실                         │
│    3. 반직관적이거나 의외의 결과                             │
│  • 카테고리 분류: cause, treatment, diet, lifestyle 등       │
└─────────────────────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────────────────┐
│  STEP 4: 블로그 글 구성                                      │
│  ─────────────────────────────────────────────────────────  │
│  • 차별화 포인트를 2~3개 섹션으로 그룹핑                      │
│  • 각 섹션에 논문 출처 매칭                                  │
│  • 스토리라인 구성 (도입 → 전개 → 정리)                      │
└─────────────────────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────────────────┐
│  STEP 5: HTML 블로그 생성                                    │
│  ─────────────────────────────────────────────────────────  │
│  • 아래 작성 규칙에 따라 최종 HTML 생성                       │
│  • Unsplash 이미지 자동 삽입                                 │
│  • output/ 폴더에 저장                                       │
└─────────────────────────────────────────────────────────────┘
                    ↓
[출력] {키워드}_{날짜}.html
```

## 2. 글 구조 규칙

### 2.1 섹션 개수
- **최소 2개, 최대 3개** (절대 4개 이상 금지)
- 섹션이 많으면 산만해지고, 적으면 깊이가 없음

### 2.2 각 섹션 분량
- **6~10문단** 이상
- 충분한 맥락과 설명 포함
- 단순 나열 금지, 이야기처럼 풀어서 작성

### 2.3 전체 구조
```
[히어로 이미지]

[제목 섹션]
- 부제: "60편 논문 분석" 등
- 메인 제목: 구어체 포함, 호기심 유발
- 배지: "최신 논문 60편 분석"

[서두 인사말] ← 필수!
- 3~5문장
- 날씨, 개인 경험, 일상적 계기 등 친근한 연결고리
- 글을 쓰게 된 배경 간략히

[본문 도입]
- 기존 상식이나 흔한 정보 언급
- "근데 정말 그럴까요?" 식의 호기심 유발

[섹션 1] + 이미지
- 핵심 발견 1~2개 깊게 다룸
- 통계 박스, 인용문 활용
- 연구 배경 → 결과 → 의미 해석 순서

[섹션 2] + 이미지
- 핵심 발견 1~2개 깊게 다룸
- 실용적 조언이나 주의사항 포함

[섹션 3] (선택)
- 추가 발견 또는 종합 정리

[하이라이트 박스]
- 핵심 내용 bullet point 정리

[마무리]
- 2~3문장
- 독자에게 말 건네는 톤

[참고 논문]
- 주요 논문 5~6개 + "외 N편"

[면책 조항]
```

## 3. 톤 & 문체 규칙

### 3.1 서두 인사말 (필수)
```
✅ 좋은 예:
"요즘 아침저녁으로 일교차가 크더라고요. 이런 날씨에는 괜히 속도
더 불편해지는 것 같습니다. 저도 한동안 역류성식도염으로 고생했던
적이 있어서, 이 주제로 논문들을 꽤 찾아봤습니다."

"며칠 전에 마트에서 오렌지주스를 집어 들다가 문득 궁금해졌습니다.
이거 진짜 건강에 좋은 걸까?"

"날씨가 슬슬 풀리니까 공원에 뛰는 사람들이 부쩍 많아졌더라고요."

❌ 나쁜 예:
"안녕하세요. 오늘은 역류성식도염에 대해 알아보겠습니다."
"이 글에서는 오렌지주스의 건강 효과를 분석합니다."
```

### 3.2 구어체 사용 위치
- **제목/헤드라인**: "그냥 뛰면 되는 줄 알았는데", "이거 잘못 알고 계실 수도 있어요"
- **문단 첫 문장**: "이게 좀 충격적인 연구였어요", "솔직히 이 부분이 제일 궁금했어요"
- **연결어**: "근데요", "그러니까", "좀 의외였는데"

### 3.3 구어체 허용/금지
```
✅ 허용:
- "~더라고요", "~거든요", "~잖아요"
- "솔직히", "근데", "좀", "꽤"
- "이게 뭐냐면요", "정리하면요"

❌ 금지 (격 떨어짐):
- "ㅋㅋ", "ㅎㅎ", "ㄷㄷ"
- "존맛", "레알", "개~"
- "~했삼", "~임ㅋ"
- 과도한 이모지 남발
```

### 3.4 전문용어 처리
```
✅ 좋은 예:
- "역류성식도염" (GERD 사용 금지)
- "최대산소섭취량" (VO2max는 괄호 안에만)
- "미토콘드리아(에너지 공장)"

❌ 나쁜 예:
- "GERD의 pathophysiology를 살펴보면..."
- "VO2max가 significant하게 improve..."
```

## 4. 시각 요소 규칙

### 4.1 이미지
- **히어로 이미지**: 1개 (상단, 300px 높이)
- **섹션 이미지**: 섹션당 1개 (선택적으로 2개)
- **소스**: Unsplash (무료, 고화질)
- **URL 형식**: `https://images.unsplash.com/photo-{ID}?w=700&h=400&fit=crop`

### 4.2 통계 박스
```html
<div class="stat-box">
    <span class="big-number">60%</span>
    <span class="stat-label">설명 텍스트</span>
</div>
```
- 핵심 수치 강조할 때만 사용
- 글 전체에서 2~3개 이내

### 4.3 인용문
```html
<div class="citation">
    "논문에서 인용한 내용..."<br><br>
    — 저널명, 연도
</div>
```
- 연구 결과 직접 인용 시 사용
- 너무 길면 요약해서 인용

### 4.4 강조 박스
- **key-point**: 핵심 정리, 팁
- **warning**: 주의사항
- **highlight**: 최종 요약

## 5. 출처 표기 규칙

### 5.1 본문 내 인용
```
출처: "논문 제목 요약", 저널명, 연도
```

### 5.2 참고 논문 섹션
```
• 논문 제목 (저널명, 연도)
• 논문 제목 (저널명, 연도)
...
외 N편
```

### 5.3 금지 사항
- PMID만 표기 (예: "출처: PMID 12345678") ❌
- DOI 링크만 표기 ❌

## 6. 콘텐츠 품질 체크리스트

### 6.1 차별화
- [ ] 기존 블로그에 없는 정보가 2개 이상 있는가?
- [ ] 구체적인 통계/수치가 3개 이상 있는가?
- [ ] "몰랐던 사실", "의외의 결과"가 포함되어 있는가?

### 6.2 구조
- [ ] 섹션이 2~3개인가?
- [ ] 각 섹션이 6문단 이상인가?
- [ ] 서두에 친근한 인사말이 있는가?

### 6.3 문체
- [ ] 제목/헤드라인에 구어체가 있는가?
- [ ] 문단 시작에 자연스러운 연결어가 있는가?
- [ ] 영어 전문용어가 한글로 풀어져 있는가?

### 6.4 시각
- [ ] 히어로 이미지가 있는가?
- [ ] 각 섹션에 이미지가 있는가?
- [ ] 통계 박스가 적절히 사용되었는가?

```
예시:
- 역류성식도염_v7_20260205.html

- 러닝_v7_20260205.html
```

## 8. 피해야 할 것들

### 8.1 구조적 문제
- ❌ 섹션 4개 이상
- ❌ 짧은 문단 나열 (프레젠테이션 스타일)
- ❌ bullet point만으로 구성

### 8.2 문체 문제
- ❌ AI가 쓴 것 같은 딱딱한 문체
- ❌ "~입니다", "~습니다"로만 끝나는 문장
- ❌ 감정 없는 백과사전식 나열

### 8.3 콘텐츠 문제
- ❌ 기존 블로그와 똑같은 뻔한 내용
- ❌ 통계/수치 없이 모호한 주장
- ❌ 출처 없는 정보

## 9. 예시: 좋은 글 vs 나쁜 글

### ❌ 나쁜 예 (프레젠테이션 스타일)
```
## 1. 원인

역류성식도염의 원인은 다음과 같습니다.

- 하부식도괄약근 이완
- 비만
- 임신
- 흡연

## 2. 증상

- 속쓰림
- 신물
- 가슴 통증
```

### ✅ 좋은 예 (이야기 스타일)
```
## 1. 속쓰림이 있다고 다 역류성식도염일까?

이 질문에서 시작해볼게요. 속이 쓰리면 자연스럽게 "역류성식도염인가?"
생각하게 되잖아요. 병원 가면 그렇게 진단받는 경우도 많고요.
근데 이게 좀 충격적인 연구가 있었습니다.

이란에서 5,561명의 내시경 결과를 분석한 대규모 연구인데요...
(계속)
```

## 10. 자동화 구현 시 참고사항

### 10.1 Claude API 프롬프트 구조
```
시스템 프롬프트:
- 이 가이드라인 전체 내용 포함
- 예시 글 포함

유저 프롬프트:
- 키워드
- 추출된 차별화 포인트 (JSON)
- 참고할 논문 목록
```

### 10.2 이미지 자동 선택
- Unsplash API 또는 미리 정의된 키워드별 이미지 매핑
- 키워드: 건강, 음식, 운동, 의료 등 카테고리별 이미지 풀

### 10.3 품질 검증
- 생성 후 체크리스트 자동 검증
- 섹션 수, 문단 수, 이미지 유무 등 자동 체크
